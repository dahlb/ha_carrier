172 def schedule_reconcile_burst(self) -> None: 

173 """(Re)start the post-write reconcile burst. 

174 

175 Right after a write the cloud's read model lags the physical unit, so 

176 its pushes — including the post-write snapshot the API layer already 

177 requests — can restate pre-write status that overwrites a genuine 

178 transition (for example a zone stuck ``idle`` while the compressor 

179 runs). The burst re-asks on ``RECONCILE_BURST_DELAYS_SECONDS`` backoff: 

180 early steps converge the UI quickly when the cloud is fast, the later 

181 steps land after the cloud's measured worst-case lag and after 

182 ``POST_WRITE_INTERCEPT_WINDOW_MINUTES`` expires, re-pulling truth once 

183 the post-write guard stops re-asserting. A newer write restarts the
